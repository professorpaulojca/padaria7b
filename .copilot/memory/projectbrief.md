# Project Brief

- **Objetivo**: Desenvolver sistema de controle geral para padaria de pequeno porte — apoiar administração, reduzir perdas operacionais, organizar informações dispersas em cadernos e rotinas informais.
- **Stakeholders**: Dono da padaria (usuário principal), funcionários (atendentes, caixa, chapeiro), clientes (fiado).
- **Escopo (in)**:
  - Estoque (ingredientes + produtos prontos + materiais de apoio)
  - Vendas (balcão, mesas, viagem, delivery/iFood)
  - Fiado (cadastro cliente, débitos, pagamentos, saldo)
  - Caixa e financeiro básico (pagamentos, vales, sangrias, fechamento)
  - Cadastros (clientes, funcionários, fornecedores, produtos, ingredientes, mesas)
  - Operação interna (ponto simplificado, manutenção, higiene, ocorrências)
  - Relatórios gerenciais simples
- **Escopo (out)**:
  - Contabilidade formal, emissão fiscal completa, integração bancária
  - Folha de pagamento completa, BI avançado, automações industriais
- **Requisitos-chave**: 20 RFs (RF01–RF20), 12 RNFs (RNF01–RNF12), 8 ROPs
- **Restrições**:
  - Usuários com baixa alfabetização digital → interface simples obrigatória
  - Projeto acadêmico → documentação rigorosa
  - Stack: React + Java Spring + PostgreSQL
