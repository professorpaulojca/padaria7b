# Workflow: Problem Analysis & Planning (Brownfield)

Quando usar:
- bug intermitente
- mudança em fluxo crítico
- alteração em mais de 2 arquivos ou >30 min

Passos:
1. **Definir o problema** (1-3 frases) + impacto.
2. **Evidências** (arquivos, logs, requests, procedures).
3. **Hipóteses** (máx. 3) e como validar cada uma.
4. **Plano mínimo** (ordem de execução) + arquivos tocados.
5. **Validação** (como provar que resolveu: teste, passos, evidência).

Saída:
- Atualizar `activeContext.md` com: problema + plano + validação.
